0. 系统定位与设计哲学

0.1 这个系统是什么、不是什么

是：一台"批量带反对派的尽调覆盖器"。它把人工只能覆盖 5-10 家公司的深度尽调流程（卡点穿透 + 证据溯源 + 魔鬼代言人 + 仓位纪律），机械化到能持续覆盖 200-800 家初筛池、20-50 家深度池，并用无幸存者偏差的全样本数据持续校准自己。

不是：十倍股预测器。十倍股在任何周期都是个位数样本，任何声称"预测十倍"的系统在统计上不可证伪、在工程上必然过拟合。

诚实的主 KPI：入场组（C4）的"5 年内最大涨幅 >= 200%（3 倍）"比例，显著高于硬门槛基线组（C1）。十倍是这个分布的右尾彩票——系统的工作是把彩票池的密度做高，并在中奖时拿得住，不是预言哪张中奖。

0.2 四个核心决策（解锁上轮全部致命缺陷的钥匙）

#问题决策
1卡点与量价的入场时点矛盾（同时满足 -> 系统性买在底部走完之后）**先资格、后时机**。卡点不参与择时，只管池子准入/逐出（季频慢腿）；量价只对池内标的开火（日频快腿）。两腿永不在同一时刻打架。
2途中 3-5 次 40-60% 回撤拿不住（任何移动止损必然洗出十倍股）**仓位状态机 E1->E2->E3，止损规则随状态切换**。侦察仓硬止损；核心仓 E3 **撤销一切百分比/均线止损**，只受 kill criteria、灾难闸门、时间止损三者管辖。
3幸存者偏差每季度全 A 快照落库做分母；C0-C6 七个 cohort（含 swarm 否决组、单腿对照组）全部跟踪前向收益；卖出后继续跟踪 12 个月测卖飞成本。
4自由度爆炸 -> 过拟合全系统参数清单化编号（param_registry），**活参数 <= 8 个**，每季度最多调 2 个，调参前必须影子运行 2 个季度。其余参数冻结。

2.1 L1 资格层（GATE，季频慢腿）

职责：回答"谁有资格被量价腿扫描"。不回答"现在买不买"。

2.1.1 硬门槛 HG1-HG7（全部可从 API 机械验证，零人工判断）

#规则阈值数据源
HG1总市值30 亿 <= mktcap <= 500 亿东财行情
HG2成长性TTM 营收同比 >= 20%（P03），**或** 最近两季营收增速逐季加速RPT_DMSK_FN_INCOME
HG3定价权痕迹毛利率 >= 25% 且 TTM 毛利率同比降幅 < 5pct同上
HG4现金流真实性近 4 季经营现金流净额之和 > 0现金流量表
HG5杠杆资产负债率 <= 60%RPT_DMSK_FN_BALANCE
HG6可信度上市 >= 365 天；非 ST/*ST；近 12 月无证监会立案universe_snapshot
HG7商誉雷商誉/净资产 <= 30%资产负债表

通过 = 7 条全过。每条的逐条布尔结果写 hard_gate_result（便于复盘"哪条门槛筛掉了后来的十倍股"——这是校准层的关键诊断数据）。预期通过数量 200-800 只；连续两季超出此区间则视为门槛失准，自动生成参数复盘提案。

2.1.2 卡点评分 gate_score（0-100，季频，需人工复核）

复用既有 stock_analysis 四维框架（irreplaceability / tech_lockin 字段），但每一分都必须挂证据：

维度分值计分规则
不可替代性0-40合格供应商数 <=2 -> 40；3-4 -> 25；>=5 -> 10。**该维度证据等级必须 >=B，否则按 >=5 家保守计 10 分**
技术锁定0-25客户认证周期 >=2 年 -> +15；存在可证的切换成本（公告/调研纪要）-> +10
价值量斜率0-20bom_analysis.value_add_pct 三年趋势：上升 -> 20；持平 -> 10；下降 -> 0
客户结构0-15绑定行业前三大客户 -> +10；第一大客户收入占比 < 50% -> +5

准入：硬门槛全过 且 gate_score >= 60（P01）且 不可替代性维度证据 >=B

逐出：gate_score < 45（P02，与准入留 15 分防抖带）或 硬门槛连续两季破 或 关键证据被证伪（evidence status='refuted'）。逐出时若该标的有持仓，自动生成 kill 复核任务（见 4.3），但逐出本身不自动卖出——卖出永远走 L4 的规则。

2.1.3 evidence_ledger 证据账本（对付约束 5：LLM 在专有事实上幻觉率最高）

每条关键主张（claim）一行，强制溯源：

  • 证据等级：A = 公司公告/招股书/年报原文（含页码）；B = 两个独立信源交叉验证（Metaso 溯源链接，同源转载不算独立）；C = 单一媒体报道；D = LLM 推断。
  • 强制规则：支撑 gate_score >= 20 分的 claim（实际上就是"供应商 <=2 家""替代窗口 >3 年"这类专有工业事实）必须 >=B 级且 verified_by='human'。D 级证据在任何计分中按不存在处理。
  • 证据过期：每条证据带 expire_date（默认 365 天）。过期自动降为 stale，对应维度分数回退保守值，并生成季度复查任务。竞争格局是流动的——去年"全球唯二"今年可能量产出第三家。
  • 复核工作流：Metaso 搜索（保留溯源链接）-> 人工打开 source_url -> 摘录原文进 source_quote -> 人工定级。LLM 可以做初筛和摘要，定级权只在人。

2.2 L2 择时层（TRIGGER，日频快腿，只扫池内）

职责：回答"池内谁现在出现了右侧确认"。不回答"这家公司好不好"（那是 L1 的事，已经答过了）。

Stage-2 突破三条件（参考 Weinstein 阶段分析，参数刻意少）：

#条件精确定义
T1 趋势模板多头排列close > MA50 > MA150 > MA200，且 MA200 今值 > 21 个交易日前值（斜率向上）
T2 量价突破创高放量close >= 52 周最高收盘价 x 95%（P05：距高 <=5%），且当日成交量 >= MA50(vol) x 1.5（P04）
T3 基底存在蓄势充分突破前存在 >= 30 个交易日（P06：6 周）的整理区，区间振幅 (max-min)/min < 25%

触发 = T1 且 T2 且 T3，写入 trigger_signal，48 小时内必须召开 L3 投决（防拖延腐化信号）。

反过滤与边界处理：

  • 突破当日涨幅 > 9%（接近涨停，实际无法成交）-> 信号标记 wait_pullback，此后 10 个交易日内回踩不破基底上沿视为有效入场窗。
  • 上市不足 250 日 -> 用上市以来最高价替代 52 周高，标记 short_history=1（cohort 分析时单独分桶，禁止混入主样本）。
  • T3 基底识别用 pattern_recognition 辅助，但判定以上述数值规则为准，形态识别只作 swarm 参考材料——形态学不进硬规则，防自由度泄漏。
  • 同日多标的触发 -> 按 gate_score 降序排队，受组合约束截断。
  • 北向资金、龙虎榜、资金流向：不出现在 T1/T2/T3 任何条件中（约束 4）。

2.3 L3 仲裁投决层（ARBITER，事件驱动）

触发：trigger_signal 新增且 48h 内。工具：vibe-trading run_swarm（bull analyst / bear analyst / risk officer / PM）。

输入材料包（自动组装，写入 swarm 的 context）：

  1. gate_score 明细 + evidence_ledger 关键证据原文摘录（含等级）
  2. trigger_signal 明细（T1/T2/T3 数值）
  3. 最近 4 季财务摘要（Kimi datasource 拉取）
  4. macro_screening 最新快照（仅作背景，不计分）
  5. 龙虎榜/资金流向（仅作背景，不计分）

结构化否决清单（bear/risk officer 不许说"风险较大"这种废话，只能引用 V1-V4）：

#否决情形可检验定义
V1证据空心化任一支撑 >=20 分的 claim 证据等级
V2kill 前兆最近 30 天内出现该标的任一 kill criteria（见 4.3）的前兆事件（如大客户公告自研、竞品中标）
V3组合约束满持仓已 8 只 / 该产业链已 40% / E1 总敞口已 6%
V4流动性不足20 日均成交额 < 1.5 亿元（E3 目标仓位按 8% x 组合规模估算，进出冲击成本 > 2% 即否决）

PM 输出（写 swarm_verdict）：

  • GO：进入 L4，开 E1 侦察仓。
  • NO-GO：引用 V1-V4 编号 + 一段论据。该标的 60 天内不再重复投决（除非触发新信号且否决原因已消除）。
  • WAIT：附复核日期（<=14 天）与复核条件。最多连续 WAIT 两次，第三次强制 GO/NO-GO。

全部 NO-GO 的标的进 C5 cohort 继续跟踪前向收益——swarm 否决组的表现是检验"反对派有没有用"的唯一标准。如果 C5 长期跑赢 C4，说明 swarm 在做反向指标，必须重构提示词。

3. 入场仲裁机制——两条腿的时点矛盾如何解开

3.1 矛盾的本质（上轮审阅指出的）

卡点识别是左侧逻辑（产业地位在股价底部就成立），量价突破是右侧逻辑（确认要等趋势走出来）。如果要求"同时满足"，等价于要求"在产业逻辑被市场充分认知之后才进场"——系统性买在主升浪中后段。

3.2 解法：先资格、后时机（Eligibility, then Timing）


时间轴 ----------------------------------------------------------------->
 [卡点成立]          [入池]              [量价触发]      [swarm GO]  [E1]
  产业事实层         季度评审              任意时点         48h内      开仓
     |<--- 可以等很久，池子不催促 --->|       |<-- 48h -->|
  卡点回答: 这家公司值得被盯着          量价回答: 市场开始认知了，现在上车
  • 卡点永不催促入场。一家公司可以在池子里躺 2 年没有任何动作——这是 feature 不是 bug（产业逻辑领先市场认知 1-3 年是常态）。
  • 量价永不背着资格开火。池外标的出现再漂亮的突破也不会产生信号——系统不做"纯技术面投机"。
  • 两腿的耦合点只有一个：池子成员名单。L1 每季度更新名单，L2 每天只扫名单内。
  • 时序上永远是"资格在前、时机在后"，且资格的有效性独立于价格位置——底部、半山腰、新高都可以入池，但只有出现 Stage-2 右侧确认才开火。买的是"右侧的早段"，放弃抄底（接受这个代价，换取胜率与可执行性）。

4.2 卖出的完整枚举（系统只有这五种卖出，不存在第六种）

  1. 侦察失败：E1 硬止损 / 90 天未确认。
  2. 降级清仓：E2 周线 MA200 破位。
  3. kill criteria：论点失效（4.3）。任何状态下都生效，E3 唯一的基本面卖出通道。
  4. 灾难闸门：组合级别动作（4.5）。
  5. 时间止损：24 个月总回报 < +20% 且最近两季营收增速均 < 15%（4.4）。
  6. （再平衡减仓不是"卖出决策"，是仓位维护。）

没有"止盈"。十倍系统设利润目标位止盈是自我否定——涨 3 倍卖掉的系统永远拿不到 10 倍。利润保护由再平衡（15%->10%）+ kill criteria 承担。

4.3 Kill Criteria（论点失效清单）

通用五条（K1-K5，所有持仓自动适用）：

#条款可检验定义检查方式
K1核心客户流失第一大客户收入占比环比降 > 15pct 且有公告/调研证实丢单财报季 Kimi + Metaso 周扫
K2替代技术落地竞品公告量产 且 获得原客户订单（两条都要，防"PPT 替代"误杀）Metaso 周扫 + evidence_ledger
K3定价权丧失毛利率连续两季下滑合计 > 8pct财报季自动计算
K4治理恶化实控人/核心技术团队变动 且 重要股东减持 > 5%公告监控
K5财务造假立案调查 / 审计非标意见公告监控。**K5 不等 5 日，次日开盘即清**

标的特化 2-3 条：swarm GO 时由 bear analyst 强制产出（写 kill_criteria 表，type='specific'），必须是可检验事件，例如"英伟达下一代平台改用 XX 方案则本标的 HBM 逻辑失效"。入场即立遗嘱——买入那天就写好什么情况下承认错误。

执行流程：cron 周检扫描 -> 疑似触发 -> 24h 内人工核实证据（写 evidence_ledger）-> 确认则 5 个交易日内分批清仓（K5 除外）-> review_log 记录全过程。

4.4 时间止损

入场满 24 个月时检查：总回报 < +20% 且 最近两季营收增速均 < 15% -> 清仓。

逻辑：十倍论点的燃料是高速成长。两年不涨可以忍（市场迟钝），但两年不涨 + 成长熄火 = 论点本身错了。只要营收还在跑，时间止损不触发——给"逻辑兑现慢"留足空间。

4.5 灾难闸门（宏观的唯一入口，约束 7 的落地）

宏观不选股（gate 权重 0）、不择时（trigger 权重 0），只保留一个组合级二元开关：

#触发动作
G1沪深300 单日 -7% 或 5 日累计 -12%组合目标仓位 x0.5：E1 全清，E2/E3 各减半。30 天后复评，复评通过则按状态机规则逐步回补
G2国家级针对持仓产业链的禁运/制裁/牌照管制公告该链持仓全部降至 E1 水平（1.5%），并对该链全部池内标的发起 kill 复核

灾难闸门动作记 trade_log(reason='gate_G1/G2')，卖出部分同样进 sold_tracking——闸门本身的成本也要被审计（如果历史数据显示 G1 每次都卖在底部，复盘会要砍掉它）。

4.6 40-60% 回撤持有手册（"什么信号让你继续持有"的正面回答）

触发：position_state.high_watermark 回撤 >= 30% 的持仓（cron 周检发现）-> 强制生成 H1-H4 checklist 任务，3 个交易日内完成，结论写 review_log。

#检查项通过标准工具
H1卡点逻辑完好供应商格局/替代窗口无恶化：evidence_ledger 无新增 refuted，Metaso 复查无 K2 前兆metaso_web_search + 人工
H2成长未熄火最近季报营收增速 >= 15% 且毛利率降幅 < 5pctKimi datasource
H3回撤是 beta 不是 alpha相对同链基准（链内池子等权指数）超额回撤 < 15pctget_market_data 计算
H4无 kill 触发kill_criteria 表全部 status='armed'（无 triggered）SQL

裁决：

  • 四条全过 -> 持有不动（E3 一股不卖）。把结论写进 review_log，下次回撤直接参考。
  • H1 或 H4 破 -> 进 kill 流程（4.3）。
  • 仅 H2 破 -> E3 降级至 E2 仓位（4%），观察一个财报季，下季恢复则补回。
  • 仅 H3 破（个股独跌但基本面查无实据）-> 持有，但标记 alpha_drawdown=1，连续两次独跌触发深度复查（市场可能知道我们不知道的事）。

这个手册就是"拿住"的工程化：回撤期间唯一允许的操作是核查论点，唯一禁止的操作是看着价格做决定。

5. 回测与校准层（L5，无幸存者偏差）

5.1 分母构建

每季度末（cron）拉取全部 A 股（含当季已退市/ST 的——退市股保留在快照里，前向收益按退市价计 -100% 或最后成交价，绝不删除）写入 universe_snapshot。这是一切统计的分母。

5.2 七个 cohort（同一只股票可同时属于多个 cohort）

Cohort定义回答的问题
C0全 A基准分布
C1仅硬门槛通过财务筛子本身值多少
C2C1 + gate_score >= 60（入池）卡点评分增量价值
C3C2 + 量价触发（无论 swarm 结论）量价腿增量价值
C4C3 + swarm GO（实际入场组）系统全链路精确率
C5C3 - C4（swarm 否决组）反对派是增益还是反指
C6入池但 5 年内量价从未触发卡点单腿对照（"好公司但市场永远没认"组）

每季 cohort 成员写 cohort_membership（记 entry_date、entry_price），月度 cron 更新 fwd_6m/12m/24m/36m/60m 前向收益与期间最大涨幅 max_runup。

5.3 核心报表（季度复盘会固定输出）

  1. 精确率表：P(60 个月内 max_runup >= 200% | cohort)，C4 vs C1 vs C0。系统存在的意义 = C4 显著 > C1。十倍命中（>=900%）单列但只作观察项（样本太小，不做决策依据）。
  2. 分桶收益分布：各 cohort 前向收益的 P10/P25/中位数/P75/P90——看分布形态不看均值（均值被右尾绑架）。
  3. 漏斗诊断：hard_gate_result 逐条统计"每条门槛各筛掉了多少后来的 3 倍股"（HG 条款的代价审计）。
  4. swarm 审计：C5 vs C4 前向收益对比 + 否决理由 V1-V4 分布。
  5. 卖飞成本表：sold_tracking 中各卖出原因（侦察失败/降级/kill/灾难/时间止损）卖出后 12 个月收益——哪类卖出规则在系统性卖飞，哪类在真救命。
  6. 持有手册审计：H1-H4 checklist 历史裁决 vs 此后 12 个月走势。

5.4 参数治理（约束 6 的落地）

  • 活参数恰好 8 个（P01-P08，见附录 A），其余全部冻结（写死在代码里，修改 = 系统大版本）。
  • 季度复盘会最多调整 2 个活参数；任何调整必须先在影子组合（不实盘，规则并行跑）运行 2 个完整季度且影子结果优于现行。
  • param_registry 记录每次变更：旧值/新值/理由/影子运行证据/生效日期。回测时按 as-of 参数重放，防"用今天的参数美化昨天的成绩"。
  • 历史回测纪律：用 backtest + factor_analysis 验证量价腿时，训练窗 2015-2021、验证窗 2022-2024、2025 起 walk-forward；财务数据按公告日对齐（防前视偏差）。

1.3 热度综合评分公式

Heat_Score = Σ (维度得分 × 维度权重)
维度得分 = Σ (子指标归一化值 × 子指标权重)

归一化方法: Min-Max标准化，将原始值映射到 [0, 100]。

热度等级: - Hotspot (≥80): 多维度共振，立即进入赛道Gate - Warm (60-79): 部分维度热，进入候选观察池 - Neutral (40-59): 中性，季度复查 - Cold (<40): 不进入Gate流程

1.4 趋势性热门 vs 周期性热门 的区分

这是最容易出错的地方——2022年锂矿就是周期性热门（后来腰斩）。

区分标准（需同时满足3条）： 1. 需求侧: TAM的CAGR ≥ 15%（非周期反弹，而是结构性增长） 2. 供给侧: 产能扩张受限于技术/牌照/资源（非"有钱就能产"） 3. 价格传导: 下游对涨价不敏感（毛利率传导通畅）

2.1 产业生命周期识别框架

  ├── 不满足 → 停止，向上聚合

核心问题: 好赛道+好公司 = 一定好投资吗？不一定——如果已经太贵。

判断"是否太贵"的三个锚：

锚 计算方法 AI算力示例 判定标准
锚1: TAM天花板法 TAM × 合理市占率天花板 × 合理PS AI训练芯片 TAM=800亿 × 40% × PS=15x → 市值天花板≈4800亿 当前市值 > 天花板70% → ⚠️贵

综合判断：

"已经太贵"的信号:
  1. PEG > 3x → 估值已透支3年以上增长
  2. 当前市值 > TAM天花板 × 70% → 空间有限
  3. 上市不满6个月且IPO估值高于一级市场最后一轮 × 3 → 一级已充分套利

"不怕贵"的条件（需同时满足）:
  1. TAM在持续扩大（3年CAGR > 30%）→ 天花板在上移
  2. 公司在扩大市场份额（份额每年+2pp以上）→ 价值捕获在加速
  3. PEG < 1.5x → 高增长消化高估值的时间在可接受范围

6.3 A股合理市值估算公式

A股合理市值 = 美股对标市值(CNY) ÷ 天花板倍数
天花板倍数 = 基准折价 × (1 + 国产替代溢价因子)

基准折价因子:
  - 流动性折价: 15-20% (美股流动性优于A股)
  - 地缘政治折价: 5-30% (取决于制裁风险)
  - 技术代差折价: 1% / 年代差 (= 差5年就折25%)

国产替代溢价因子:
  - 国产化率越低(<20%)需求越刚性 → 溢价越高
  - 政策优先级越高(国家级) → 溢价越高

附录B: 现有Gate体系修改要点

以下是对现有 hard_gate_result 和 gate_score 表的具体修改建议。

B.1 hard_gate_result 修改

当前状态:

hg1-hg7 硬门槛: 固定值, 所有股票用一个标准
passed: 全部通过即进入M1

修改后:

hg1(成长性): ≥ track_gate_result.benchmark_rev_growth  (而非写死15%)
hg3(毛利率):  ≥ track_gate_result.benchmark_gross_margin  (而非写死30%)
hg7(估值):    ≤ track_gate_result.benchmark_pe_ceiling     (而非不限制)

新增字段: track_name, track_tier, track_gate_snap_date
  → 支持追溯"这只股票是在哪个赛道Gate框架下通过的"

B.2 gate_score 修改

当前状态:

irreplace_score + lockin_score + value_slope_score + customer_score = total

修改后:

每个评分维度 × track_weight(该节点在产业链中的权重)

irreplace_score × node_weight(如GPU芯片权重0.20)
  → 使用 chain_composition_detail 中的 bottleneck_severity 和 localization_rate

lockin_score × node_weight
  → 使用 chain_composition_detail 中的 competitive_moat_type 和 cr3_concentration

value_slope_score 
  → 使用 chain_composition_detail 中的 value_trend 和 value_pct_in_chain

customer_score
  → 使用 industry_chain 中的 downstream 节点和 indirect_beneficiaries

B.3 candidate_pool 修改

当前状态: 直接从 hard_gate_result 中 passed=1 的股票进入候选池。

修改后: 增加一道赛道预筛选

candidate_pool 中增加字段:
  track_name TEXT,
  track_tier TEXT,
  track_node TEXT,           -- 该股票所属产业链节点
  track_node_weight REAL     -- 该节点的Gate权重

只有当 track_gate_result.pass_for_stock_gate=1 的赛道中的股票才能进入候选池。

B.4 赛道复查周期

S级赛道: 月度复查 (如AI算力)
A级赛道: 季度复查
B级赛道: 季度复查 (关注向上跃迁)
C/D级赛道: 半年复查

复查内容: 1. 热度指数是否保持? 2. 产业链结构是否有重大变化(新节点出现/旧节点萎缩)? 3. 美股对标天花板是否有变化(技术代差缩小/扩大)? 4. 政策信号是加强还是减弱?

总结: 赛道Gate的主要决策逻辑

输入: 待评估赛道
  ↓
T0 Gate (硬门槛):
  ✓ 上市公司≥3家? → NO → ❌ 淘汰
  ✓ TAM≥100亿?    → NO → ❌ 淘汰
  ✓ 竞争可辨识?   → NO → ❌ 淘汰
  ↓
Heat Score (热度评分):
  资金面(25%)+基本面(40%)+政策面(20%)+产业面(15%)
  ↓ ≥60 通过
  ↓
Lifecycle Check (生命周期):
  concept → ❌ 太早
  growth  → ✅ 最佳 (S级候选)
  maturity → ✅ A级 (关注龙头)
  decline → ❌ 不投
  ↓
US Benchmark Check (美股对标):
  天花板充足? 技术代差在缩小? → bias upward
  天花板有限? 代差在扩大?     → bias downward
  ↓
输出: Track Tier (S/A/B/C/D) + 进入M0的个股Pool

本文档中标注为 assumption 的判断基于公开数据和行业观察的合理推断，非精确量化结论。所有阈值和权重均为初始设定，需通过回测校准。
